跨域

同源策略

同源策略(Same-Origin Policy)最早由 Netscape 公司提出，是浏览器的一种安全策略。

同源: 协议、域名、端口号 必须完全相同。即目标请求资源和网页资源都是来自于这台服务器的这一个服务。

违背同源策略就是跨域。单台服务器的服务是有上限的，性能是有瓶颈的，所以这时候需要加更多的计算机，更多的服务器来提供服务

Ajax默认要遵循同源策略

ES6

ECMAScript新特性

1. let声明变量以及声明特性

let 关键字用来声明变量，使用 let 声明的变量有几个特点:

  1. 不允许重复声明
  2. 块儿级作用域
  3. 不存在变量提升
  4. 不影响作用域链

应用场景:以后声明变量使用 let 就对了

案例分析

需求：有三个div，被点击的那个div背景颜色变为pink

这里如果用var去声明变量就会报错，因为var没有块级作用域，这里是全局作用域

因为var没有块级作用域，所以 i 这个值一直都是在全局中存在的

而let声明的三个i，他们只在他们的代码块中执行，互不影响

2. const声明常量以及特点

const 关键字用来声明常量，const 声明有以下特点 1) 声明必须赋初始值
2) 标识符一般为大写
3) 不允许重复声明
4) 值不允许修改
5) 块儿级作用域
注意: 对象属性修改和数组元素变化不会出发 const 错误 应用场景:声明对象类型使用 const，非对象类型声明选择 let

3. 变量的解构赋值

ES6 允许按照一定模式，从数组和对象中提取值，对变量进行赋值，这被称
为解构赋值。

注意:频繁使用对象方法、数组元素，就可以使用解构赋值形式

4.模板字符串

模板字符串(template string)是增强版的字符串，用反引号(`)标识，特点:

  1. 字符串中可以出现换行符
  2. 可以使用 ${xxx} 形式输出变量

注意:当遇到字符串与变量拼接的情况使用模板字符串

5.简化对象写法

ES6 允许在大括号里面，直接写入变量和函数，作为对象的属性和方法。这
样的书写更加简洁

注意:对象简写形式简化了代码，所以以后用简写就对了

6.箭头函数

箭头函数的注意点:

  1. 如果形参只有一个，则小括号可以省略
  2. 函数体如果只有一条语句，则花括号可以省略，函数的返回值为该条语句的
    执行结果
  3. 箭头函数 this 指向声明时所在作用域下 this 的值
  4. 箭头函数不能作为构造函数实例化
  5. 不能使用 arguments

注意:箭头函数不会更改 this 指向，用来指定回调函数会非常合适

9.Symbol

9.1Symbol 基本使用

ES6 引入了一种新的原始数据类型 Symbol，表示独一无二的值。它是 JavaScript 语言的第七种数据类型，是一种类似于字符串的数据类型。
Symbol 特点

  1. Symbol 的值是唯一的，用来解决命名冲突的问题
  2. Symbol 值不能与其他数据进行运算
  3. Symbol 定义的对象属性不能使用 for…in 循环遍历，但是可以使用
    Reflect.ownKeys 来获取对象的所有键名

注: 遇到唯一性的场景时要想到 Symbol

9.2 Symbol内置属性

isConcatSpreadable等这些属性都是symbol里面的属性，而Symbol.isConcatSpreadable这个整体又作为对象的属性去设置，来改变对象在特定场景下的属性；即扩展对象属性

10.迭代器

js里面的Iterator接口就是对象里的一个属性，属性的名字叫做Symbol.Iterator

遍历器(Iterator)就是一种机制。它是一种接口，为各种不同的数据结构提
供统一的访问机制。任何数据结构只要部署 Iterator 接口，就可以完成遍历操作。

  1. ES6 创造了一种新的遍历命令 for…of 循环，Iterator 接口主要供 for…of 消费
  2. 原生具备 iterator 接口的数据(可用 for of 遍历)
    a) Array
    b) Arguments
    c) Set
    d) Map
    e) String
    f) TypedArray
    g) NodeList
  1. 工作原理
    a) 创建一个指针对象，指向当前数据结构的起始位置
    b) 第一次调用对象的 next 方法，指针自动指向数据结构的第一个成员
    c) 接下来不断调用 next 方法，指针一直往后移动，直到指向最后一个成员
    d) 每调用 next 方法返回一个包含 value 和 done 属性的对象

注: 需要自定义遍历数据的时候，要想到迭代器。

12.Promise(面试常问)

Promise 是 ES6 引入的异步编程（主要指一些io的代码，文件io、数据库io 、网络请求）的新解决方案（解决回调地狱的问题）。语法上 Promise 是一个构造函数，可以实例化对象， 用来封装异步操作并可以获取其成功或失败的结果。

  1. Promise 构造函数: Promise (excutor) {}
  2. Promise.prototype.then 方法
  3. Promise.prototype.catch 方法

12.1 Promise基本语法

Promise对象有三种状态：初始化、成功、失败

得到数据以后，我们可以调用resolve和reject这两个函数来改变promise对象的状态

Promise对象的then方法接收两个参数，并且两个参数都是函数类型的值

成功的形参一般叫value，失败的形参一般叫reason

调用resolve表示他成功，那么then方法就会执行它的第一个回调函数代码

调用reject表示失败，那么then方法就会执行它的第二个回调函数代码

12.4 Promise 的then 方法

调用then方法。then方法返回结果是Promise对象

1.如果回调函数中的返回结果是非Promise类型的属性，状态为成功，返回值为对象成功的值

若不写return，默认返回结果是undefined，非Promise类型的属性，也返回成功，返回值为undefined

2.如果回调函数中返回的结果是Promise值，这个Promise的状态决定了then方法返回Promise的对象的状态。这个Promise成功，then方法返回的也是成功；并且，这个Promise成功的值就是then方法返回Promise对象的值。这个Promise失败，then方法返回的也是失败；并且，这个Promise失败的值就是then方法返回Promise对象的值

3.如果抛出错误 ，那么这个状态也是Promise对象的失败的状态，错误值是抛出的那个值

then方法可以链式调用

13集合介绍与API

ES6 提供了新的数据结构 Set(集合)。它类似于数组，但成员的值都是唯 一的，集合实现了 iterator 接口，所以可以使用『扩展运算符』和『for…of…』进 行遍历，集合的属性和方法:

  1. size 返回集合的元素个数
  2. add 增加一个新元素，返回当前集合
  3. delete 删除元素，返回 boolean 值
  4. has 检测集合中是否包含某个元素，返回 boolean 值
  5. clear 清空集合，返回 undefined

14.MAP

ES6 提供了 Map 数据结构。它类似于对象，也是键值对的集合。但是“键” 的范围不限于字符串，各种类型的值(包括对象)都可以当作键。Map 也实现了 iterator 接口，所以可以使用『扩展运算符』和『for…of…』进行遍历。Map 的属 性和方法:

  1. size 返回 Map 的元素个数
  2. set 增加一个新元素，返回当前 Map
  3. get 返回键名对象的键值
  4. has 检测 Map 中是否包含某个元素，返回 boolean 值
  5. clear 清空集合，返回 undefined

15.class类

ES6 提供了更接近传统语言的写法，引入了 Class(类)这个概念，作为对 象的模板。通过 class 关键字，可以定义类。基本上，ES6 的 class 可以看作只是 一个语法糖，它的绝大部分功能，ES5 都可以做到，新的 class 写法只是让对象 原型的写法更加清晰、更像面向对象编程的语法而已。
知识点:

  1. class 声明类

  2. constructor 定义构造函数初始化

  3. extends 继承父类

  4. super 调用父级构造方法

  5. static 定义静态方法和属性

  6. 父类方法可以重写

  1. ```javascript
    <script>
    //手机
    function Phone(brand, price){
        this.brand = brand;
        this.price = price;
    }

    //添加方法
    Phone.prototype.call = function(){
        console.log("我可以打电话!!");
    }

    //实例化对象
    let Huawei = new Phone('华为', 5999);
    Huawei.call();
    console.log(Huawei);

    //class
    // 关键字 类名 花括号
    class Shouji{
        //构造方法 名字不能修改
        // 当我们使用new 类名时，就会自动去执行实例对象上的constructor方法
        constructor(brand, price){
            this.brand = brand;
            this.price = price;
        }

        //方法必须使用该语法, 不能使用 ES5 的对象完整形式
        call(){
            console.log("我可以打电话!!");
        }
    }

    let onePlus = new Shouji("1+", 1999);

    console.log(onePlus);
</script>

实例对象和函数对象的属性是不相通的，函数对象的属性是属于函数对象的并不属于实例对象，对于这样的属性我们把它称为静态成员（放到面向对象里面，这个属性是属于类的而不属于实例对象）

对于static标注的方法，它属于类，而不属于实例对象；声明关键字就是static

15.3 class 的get和set

get（当对一个属性进行获取时执行）和set（当对一个属性进行改变时执行）方法可以对对象的属性进行方法的绑定

正常来说class没有构造函数是合法的，构造函数并不是必须要有，没有也可以

而且这个函数的返回值就是这个属性的值

get通常对对象的动态属性进行封装，这个属性是变化的（例如求总数、平均数），会随着数据的变化而变化，并不是固定的，这时候用get通过一出个函数去动态的计算结果就非常的方便

set可以添加更多的控制和判断，例如做赋值的判断，如果传进来的是字符串就赋值，否则不赋值

18.模块化

模块化是指将一个大的程序文件，拆分成许多小的文件(模块)，然后将小文件组合起来。

18.1. 模块化的好处

模块化的优势有以下几点:

  1. 防止命名冲突（例如在a.js这个文件里面声明一个变量叫data，在b.js声明一个变量也叫data这两个变量互不影响）

  2. 代码复用（把一些功能代码封装成一个文件，对外暴露一个接口）

  1. 高维护性（a.开发人员对文件的修改不会产生过多的冲突 ，每人修改不同的文件，文件之间不会产生过多的冲突； b.升级只需要对某一个模块或者某一些模块升级即可，免去对大多的文件做更改和变动）

18.2. 模块化规范产品

ES6 之前的模块化规范有:

  1. CommonJS （规范，一些纸面上的东西；每个规范都有对应的实现的产品）=> NodeJS、Browserify（用的是Common.js规范）
  1. AMD => requireJS
  2. CMD => seaJS

3、4是针对浏览器端的

18.3. ES6 模块化语法

模块功能主要由两个命令构成:export 和 import。

export 命令用于规定模块的对外接口（在这个文件模块中，哪些数据需要暴露就在这个要暴露的数据前面加export）
import 命令用于输入其他模块提供的功能（用于导入）

JWT

什么是JWT

JWT（JSON WEB TOKEN）是通信双方之间以 JSON对象的形式安全传递信息的方法。

其实可以理解为使用非对称算法来进行前后端校验。

JWT 的组成

JWT 由三部分组成

头部-Header

然后按照此规则将头部信息进行base64编码，构成JWT第一部分

载荷-payload

payload 就是存放有效信息的地方

payload 中有一些参数字段是建议使用的 （仅列出几个）

参数 含义
iat jwt的签发时间
exp jwt的过期时间，这个过期时间必须要大于签发时间
nbf 定义在什么时间之前，该jwt都是不可用的

JWT规定了7个官方字段：

  • iss(issuer)：签发人
  • exp (expiration time)：过期时间
  • sub (subject)：主题
  • aud (audience)：受众
  • nbf (Not Before)：生效时间
  • iat (Issued At)：签发时间
  • jti (JWT ID)：编号

比如来定义一个payload

payload 会进行base64编码，构成JWT第二部分

签证-Singnature

Signature 部分是对前两部分的签名，防止数据篡改。

需要指定一个密钥（secret）。这个密钥只有服务器才知道，不能泄露给用户。然后，使用 Header 里面指定的签名算法（默认是 HMAC SHA256）。

signature 是JWT的第三部分。

将以上三部分拼接起来（中间用.分隔），就是最后的JWT。

  • 允许用户从NPM服务器下载别人编写的第三方包到本地使用。
  • 允许用户从NPM服务器下载并安装别人编写的命令行程序到本地使用。
  • 允许用户将自己编写的包或命令行程序上传到NPM服务器供别人使用。
